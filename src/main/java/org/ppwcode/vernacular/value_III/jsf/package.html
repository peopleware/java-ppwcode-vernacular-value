In Web 1.0 applications, objects emulating values are to be rendered as Strings to work in HTML, and need to be interpreted fromStrings submitted from a web page. In JSF, this is done through {@link javax.faces.convert.Converter Converters}, which have to be registered in the faces-config.xml file. This package adds support to make the implementation of such {@link javax.faces.convert.Converter Converters} easy for value types that follow our value vernacular, at least up to a point.
